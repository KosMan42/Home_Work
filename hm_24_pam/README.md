# Ограничение доступа пользователей по времени с использованием PAM

## Окружение

Работа выполнена на виртуальных машинах:

- Vagrant 2.4.9
- VirtualBox 7.2.6a
- Базовый образ: ubuntu/jammy64

## Текст задания
Ограничить доступ к системе для всех пользователей, кроме группы администраторов, в выходные дни (суббота и воскресенье), за исключением праздничных дней.

## Описание решения
В рамках задания реализовано ограничение доступа пользователей через PAM с использованием модуля pam_exec. Для этого создан скрипт /usr/local/bin/login.sh, который проверяет текущий день недели и принадлежность пользователя к группе admin.

Если день — суббота или воскресенье, доступ разрешён только пользователям из группы admin. В остальные дни доступ разрешён всем пользователям.

## Используемые команды

Создание пользователей без домашних директорий:
useradd otusadm
useradd otus

Назначение паролей:
echo "otusadm:Otus2022!" | chpasswd
echo "otus:Otus2022!" | chpasswd

Создание группы и добавление пользователей:
groupadd admin
usermod -aG admin otusadm
usermod -aG admin root

Проверка состава группы:
grep admin /etc/group

Подключение PAM-скрипта:
sed -i '/@include common-auth/i auth required pam_exec.so debug /usr/local/bin/login.sh' /etc/pam.d/sshd

Включение парольной аутентификации SSH:
rm -f /etc/ssh/sshd_config.d/60-cloudimg-settings.conf

Создание override-конфига:
cat > /etc/ssh/sshd_config.d/99-local.conf <<EOF
PasswordAuthentication yes
KbdInteractiveAuthentication no
UsePAM yes
EOF

## Особенности реализации
Для реализации ограничения использован модуль pam_exec, так как pam_time неудобен при работе с локальными группами.

Скрипт login.sh выполняется при аутентификации пользователя и возвращает код:
0 — разрешить доступ
1 — запретить доступ

Учитывались особенности Ubuntu:
- конфигурация SSH разбита на директорию sshd_config.d
- файл 60-cloudimg-settings.conf переопределяет параметры и должен быть удалён
- сервис SSH называется ssh, а не sshd

## Заметки
В выходные:
- пользователь otusadm (в группе admin) имеет доступ
- пользователь otus доступ не получает

В будние дни доступ разрешён всем пользователям.

Ошибка PasswordAuthentication no возникала из-за конфигурации cloud-init.
Ошибка Connection refused возникала при некорректной конфигурации SSH.
